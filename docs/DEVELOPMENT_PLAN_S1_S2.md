
# 통합 개발 계획: NOVA — Sprint 1 & 2

> **연관 문서**: `DEVELOPMENT_PLAN_S3_S4.md` · `DEVELOPMENT_PLAN_SUMMARY.md`
> **기간**: Week 1–6 (6주)
> **범위**: 모바일 PWA 수집 → 데스크탑 Electron 확장 & 조직화 완성

---

## 1. 요약 (Summary)

Sprint 1은 **PWA만으로 베타 출시 가능한 수준**을 목표로 한다.
모바일 카메라 촬영, iOS/Android 공유 시트, AI 자동 태깅을 구현하여
사용자가 어디서든 레퍼런스를 1클릭으로 수집할 수 있게 한다.

Sprint 2는 **데스크탑 Electron 앱을 본격 구축**한다.
로컬 파일 드래그&드롭, 브라우저 확장 6종 캡처, 계층형 폴더와 태그 시스템을 완성하여
수집한 에셋을 즉시 조직화할 수 있는 환경을 만든다.

## 2. 기술 스택

| 레이어 | 선택 | 근거 |
| :--- | :--- | :--- |
| **모바일 앱** | PWA (React 18 + Vite) | 별도 앱 설치 없이 카메라 API 사용 가능 |
| **데스크탑 앱** | Electron 30 + Node.js 20 LTS | macOS/Windows 크로스플랫폼, Sprint 2에서 PWA 코드베이스 포팅 |
| **UI 프레임워크** | React 18 + TypeScript 5 | 데스크탑/PWA 컴포넌트 공유 |
| **상태 관리** | Zustand | Electron·PWA 공용 상태 관리 |
| **스타일링** | Tailwind CSS 4 | 다크/라이트 모드, 3단계 반응형 |
| **로컬 DB (데스크탑)** | SQLite (`better-sqlite3`) | 메타데이터 저장 |
| **로컬 스토리지** | OPFS | 브라우저 내 원본 파일 고속 I/O |
| **이미지 처리** | Sharp + `node-vibrant` (데스크탑) / Web Worker (PWA) | 썸네일·주조색·팔레트 추출 |
| **AI 태깅** | Supabase Edge Function + TensorFlow.js / CLIP | 색상·구도·오브젝트 온라인 분석, 오프라인 폴백: MobileNet |
| **백엔드** | Supabase (PostgreSQL + Realtime + Auth + Storage) | 실시간 동기화, RLS 보안 |
| **브라우저 확장** | Manifest V3 (Chrome/Firefox/Edge 통합, Safari 별도) | Safari 심사 2~4주 선제 제출 필요 |
| **빌드/배포** | electron-builder (DMG + NSIS) / Vercel or Cloudflare Pages (PWA) | |

## 4. Sprint 1 — 모바일 MVP 베타 (Week 1–3)

### 4.1 목표

> "PWA만으로 베타 출시 가능한 수준을 달성한다."

| 지표 | 목표값 |
| :--- | :--- |
| AI 태그 정확도 (로컬 MobileNet) | **≥ 70%** |
| 촬영 → 로컬 저장 완료 | **< 1초** |
| 오류율 (모든 저장 경로) | **< 1%** |

### 4.2 아키텍처 결정

#### 4.2.1 로컬 저장 구조

- 촬영·수신 즉시 `OPFS`에 원본 저장, `IndexedDB`에 메타데이터 저장 → 네트워크 상태와 무관하게 항상 동작.
- Sprint 3에서 Supabase 연동 시, 로컬 에셋이 `syncStatus: 'local_only'`로 분류되어 최초 동기화 대상이 됨.

#### 4.2.2 AI 태깅 파이프라인

```
이미지 저장
  └─ 온디바이스 MobileNet (Web Worker)
       └─ 색상·구도·오브젝트 분류 + node-vibrant 주조색 5종 팔레트 추출
```

> Sprint 3에서 Supabase Edge Function(TensorFlow.js / CLIP) 연동 후 태그 품질 고도화 예정.

#### 4.2.3 PWA 설정

- `manifest.json`: `display: standalone`, `share_target` 등록 (iOS/Android 공유 시트 수신).
- 카메라: `MediaDevices.getUserMedia({ video: true })`.
- 파일 접근: `showOpenFilePicker()` 기본, 폴백 `<input type="file">`.

### 4.3 사용자 스토리별 구현 명세

#### US-1-01: 라이브러리 생성 & 온보딩

- 앱 첫 실행 시 온보딩 모달 자동 표시.
- 온보딩 완료 후 로컬 라이브러리 자동 생성 (OPFS + IndexedDB).
- 빈 라이브러리 화면으로 진입, "첫 에셋 추가" 가이드 표시.
- 온보딩은 언제든 재실행 가능.
- Sprint 3에서 Supabase 연동 시 로컬 라이브러리가 클라우드와 연결됨.

#### US-1-02: 모바일 PWA 카메라 촬영 & AI 자동 분류

| 항목 | 상세 |
| :--- | :--- |
| **PWA 설치** | 모바일 브라우저에서 "홈 화면에 추가"로 앱처럼 사용 |
| **카메라** | 앱 내 직접 실행, 촬영 즉시 저장 + AI 태그 적용 |
| **AI 태그** | `#color/` 주조색 5종 · `#comp/` 구도 · `#obj/` 오브젝트 대분류 · `#style` · `#detail` · `#palette` · `#project` |
| **갤러리** | 기기 갤러리에서 복수 선택 가져오기 |
| **인박스** | 모든 수집 이미지는 라이브러리 최상단 인박스에 우선 저장, 미분류 수량 배지 표시 |
| **UI 구성** | 풀스크린 뷰파인더 · 셔터 버튼 · 인박스 배지 · 최근 저장 썸네일(우하단) |
| **지원 환경** | iOS Safari 16+, Android Chrome 76+ |

#### US-1-03: iOS Share Sheet / Android 공유하기

- iOS/Android 공유 메뉴에 NOVA 항목 표시.
- 이미지·URL 수신 즉시 라이브러리 저장 + AI 자동 태그.
- URL 공유 시 추천 메타데이터(페이지 타이틀·썸네일·도메인) 함께 저장.

#### US-1-04: 로컬 영구 저장 보장

> Sprint 1 & 2는 Supabase 없이 로컬 단독 동작. 모든 에셋은 로컬에만 저장되며, Sprint 3 Supabase 연동 시 최초 동기화 대상이 된다.

- 촬영·수신 즉시 OPFS에 원본 저장, IndexedDB에 메타데이터 저장.
- 앱 종료 후 재시작해도 저장된 에셋이 유지됨 (영구 저장 검증).
- 로컬 저장 실패 시 오류 토스트 + 재시도 버튼 제공.
- 저장 용량 부족 시 경고 토스트 표시.

### 4.4 Sprint 1 완료 조건 (Definition of Done)

- [ ] iOS Safari / Android Chrome 카메라 촬영 → OPFS 로컬 저장 E2E 성공

- [ ] 앱 재시작 후 저장된 에셋 유지 확인 (로컬 영구 저장)
- [ ] 모든 저장 경로에서 오류율 < 1% (QA 체크리스트 통과)
- [ ] 온보딩 플로우 E2E 정상 동작

## 5. Sprint 2 — 데스크탑 확장 & 조직화 (Week 4–6)

### 5.1 목표

> "데스크탑 수집 기능을 완성하고 Electron 앱을 본격 구축한다."

| 지표 | 목표값 |
| :--- | :--- |
| 폴더·태그·스마트폴더 조작 응답 | **< 200ms** |
| 스마트 폴더 조건 변경 후 목록 갱신 | **< 3초** |
| 태그 20,000개 환경 자동완성 응답 | **< 100ms** |
| 드래그&드롭 후 썸네일 그리드 표시 | **< 3초** |

### 5.2 아키텍처 결정

#### 5.2.1 Electron 구조

```
Electron Main Process
  ├─ Native File System 접근 (감시 폴더, 드래그&드롭)
  ├─ Sharp (썸네일 생성 Batch)
  └─ SQLite (메타데이터 로컬 캐시)

Electron Renderer Process
  └─ React 앱 (PWA 코드베이스 포팅)
       └─ Zustand 상태 관리 (로컬 전용 — Supabase Client 없음)

※ Supabase 연동은 Sprint 3 HybridAssetRepository에서 시작.
```

#### 5.2.2 감시 폴더 (Monitored Folder)

- `chokidar`로 지정 디렉토리 파일 변경 감지.
- 새 파일 추가 시 자동 인박스 유입 → AI 태깅.
- 폴더별 기본 태그·대상 폴더 사전 설정 가능.

#### 5.2.3 브라우저 확장 아키텍처

```
Content Script
  └─ 페이지 이미지 감지 / 드래그 이벤트 캡처
        │
Background Service Worker (Manifest V3)
  └─ 캡처 처리 → Electron 로컬 HTTP API (localhost) 호출 → SQLite 저장
        │
Popup UI
  └─ 저장 완료 토스트, 대상 폴더 선택
```

> Electron Main Process가 로컬 HTTP 서버를 열어 브라우저 확장의 저장 요청을 수신한다.
> Supabase 업로드는 Sprint 3 동기화 시점에 처리됨.

#### 5.2.4 스마트 폴더 조건 엔진

- 조건 JSON 스키마: `{ operator: 'AND'|'OR', rules: [{ field, op, value }] }`.
- 에셋 저장/수정 시 관련 스마트 폴더 조건 재평가 → 실시간 업데이트.

### 5.3 사용자 스토리별 구현 명세

#### US-2-00: 데스크탑 앱 메인 레이아웃 & 기본 화면

#### US-2-01: 로컬 파일 드래그&드롭 저장 & 감시 폴더

| 항목 | 상세 |
| :--- | :--- |
| **드래그&드롭** | 단일 파일 및 폴더(하위 파일 포함) 드롭 지원 |

#### US-2-02: 브라우저 확장 7종 캡처 & 수집 모드

| 모드 | 동작 |
| :--- | :--- |
| **Batch Save** | 현재 페이지 모든 이미지 체크박스 선택 후 일괄 저장 |
| **Auto-Detect** | 스크롤·클릭 중 새로 나타나는 이미지 자동 감지·수집 |
| **Capture Area** | 마우스 드래그로 영역 지정 → 스크린샷 즉시 저장 |
| **Capture Element** | HTML 요소(버튼·카드 등) 클릭 → 해당 요소만 캡처 |
| **Capture Visible** | 현재 뷰포트 1:1 캡처 |
| **Full Page** | 전체 페이지 1장 이미지 저장 (최대 높이 제한 있음) |
| **URL 북마크** | 이미지 없이 URL·페이지 타이틀·파비콘·도메인만 메타데이터로 저장 |

- 저장 시 출처 URL이 에셋 메타데이터에 자동 기록.
- 저장 완료 시 우하단 토스트, 대상 폴더를 확장 팝업에서 직접 선택 가능.

#### US-2-03: 웹 이미지 드래그 자동 저장

- 확장 활성화 상태에서 웹 이미지 드래그 시 화면 우측에 저장 드롭존 오버레이 표시.
- 드롭존에 놓으면 원본 해상도 이미지 자동 저장 + 출처 URL 기록.
- 드래그 중 ESC → 오버레이 사라짐, 저장 취소.

#### US-2-04: 클립보드 붙여넣기 저장

- 앱 포커스 상태에서 `Ctrl+V` / `Cmd+V` → 클립보드 이미지 라이브러리 저장.
- 클립보드에 이미지 없을 경우 안내 토스트.
- 저장 파일명 자동 생성, 저장 전 네이밍 모달 표시 (확인 없이 기본값 저장 옵션 가능).

#### US-2-05: 폴더 & 중첩 폴더 시스템

- 폴더 트리 표시, 최대 5단계 중첩 지원.
- 폴더 커스텀: 14종 아이콘 × 7가지 컬러 라벨 조합.
- 폴더 간 에셋 이동 드래그&드롭 지원.
- 폴더 삭제 시 에셋은 삭제되지 않고 '미분류' 상태로 이동.
- 폴더 내 에셋 수 표시.

#### US-2-06: 폴더 자동태그

- 폴더 설정에서 '자동 태그' 항목에 태그 1개 이상 설정 가능.
- 에셋이 해당 폴더(하위 폴더 포함, 옵션)로 이동되면 설정된 태그 자동 추가.
- 자동 적용된 태그는 수동 제거 가능.
- 설정 변경 시 기존 에셋 소급 적용 여부 선택 가능.
- 자동태그 폴더는 시각적으로 식별 가능.

#### US-2-07: 다중 태그 & 태그 그룹

| 항목 | 상세 |
| :--- | :--- |
| **입력** | 에셋 인스펙터에서 복수 입력, 자동완성 지원 |
| **추천** | 입력 시 기존 태그 목록에서 추천 |
| **8개 표준 그룹** | `#color/` · `#comp/` · `#obj/` · `#detail/` · `#style/` · `#palette/` · `#project/` · `#source/` |
| **사용자 정의 그룹** | 추가 가능 |
| **사이드바** | 그룹별 펼치기/접기 |
| **성능** | 태그 20,000개 환경에서 자동완성 응답 100ms 이내 |

#### US-2-08: 스마트 폴더 (Should)

- 조건 설정: 태그 포함/제외, 파일 포맷, 별점, 생성일 범위, 컬러 라벨.
- AND/OR 논리 연산자 복합 조건 지원.
- 스마트 폴더 내 에셋 실시간 업데이트.
- 일반 폴더와 동일 위치 표시, 시각적으로 구분.

#### US-2-09: 별점 & 컬러 라벨

- 에셋별 1~5점 별점, 키보드 숫자키(1~5)로 즉시 적용.
- 7가지 컬러 라벨(빨강·주황·노랑·초록·파랑·보라·회색) 중 하나 지정.
- 라벨 색상이 에셋 썸네일에 표시.
- 별점·컬러 라벨 필터 조건으로 사용 가능.

#### US-2-10: F키 카테고리 폴 & 분류 단축키

- 에셋 선택 후 `F` 키 → 폴더 선택 팔레트 오픈, 키보드로 폴더 검색·이동.
- 팔레트에서 오타가 있어도 매칭 폴더 즉시 표시 (퍼지 검색).
- `Alt+드래그` → 에셋 복사(원본 유지)하여 다른 폴더에 저장.
- `Shift+D` → 마지막 이동 폴더에 반복 분류.
- 다중 선택 후 `F` → 선택 에셋 일괄 이동.
- 에셋 상세 화면에서도 동일한 폴더 선택 팔레트 사용 가능.

#### US-2-11: 태그 복사 & 일괄 태그 적용

- `Cmd+Shift+C` / `Ctrl+Shift+C` → 선택 에셋의 태그 목록 복사.
- 다른 에셋 선택 후 '태그 붙여넣기'로 동일 태그 세트 즉시 적용.
- 다중 선택된 에셋에 태그 추가 → 선택된 모든 에셋에 일괄 적용.
- 특정 태그를 가진 에셋 전체 선택 가능.

#### US-2-12: 일괄 이름 변경 (Batch Rename, Should)

| 파라미터 | 설명 |
| :--- | :--- |
| `%N` | 시리얼 번호 (시작값 지정 가능) |
| `*` | 원본 파일명 유지 |
| `%D` | 가져온 날짜 |
| `%M` | 수정 시각 |
| `%B` | 생성 시각 |
| `%F` | 소속 폴더명 |

- 날짜 형식: YMD / YM / Y + HM / HMS 조합.
- Find & Replace: 파일명 내 특정 문자열 치환.
- 텍스트 형식: 변환 없음 / UPPER / lower / Title.
- 변경 전 실시간 미리보기 제공.

#### US-2-13: 저장 시 메모·맥락 자동+수동

| 기능 | 상세 |
| :--- | :--- |
| **자동 메타 수집** | 출처 URL·도메인, 페이지 타이틀, 저장 일시, EXIF(촬영 위치·시간), 파일 포맷·크기 |
| **AI 자동 맥락 요약** | Supabase Edge Function 필요 → Sprint 3 이후 구현 |
| **수동 메모** | 에셋 인스펙터에서 직접 텍스트 메모 입력 (저장 중 빈 상태로 저장) |
| **메모 검색 연동** | 메모 텍스트는 키워드 전문 검색(US-4-01) 대상에 포함 |
| **커스텀 필드** | Location, Assignee, 프로젝트명 등 사용자 정의 필드 추가 가능, 필드별 검색·필터 지원 |
