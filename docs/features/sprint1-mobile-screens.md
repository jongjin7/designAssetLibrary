# Sprint 1: 모바일 화면 구성 명세 (Mobile UI Screen Specification)

Sprint 1 모바일(PWA) 앱의 각 화면(Screen)별 UI 구성 요소, 레이아웃 구조, 내비게이션 흐름을 정의합니다. 기능 명세([sprint1-mobile.md](sprint1-mobile.md))를 기반으로 실제 화면 단위의 컴포넌트 배치와 인터랙션을 구체화합니다.

Sprint 1 mobile UI consists of **3 primary screens**: Library, Camera Capture, and Profile. All screens share a fixed `BottomTabs` navigation and follow a single-column layout (`< 768px`). Search is integrated into the Library screen with a detailed filter option.

---

## 1. 내비게이션 구조 (Navigation Structure)

### 1.1 전체 화면 흐름

```text
                    ┌─────────────────┐
                    │   Splash Screen  │
                    │  (auth check)    │
                    └────────┬────────┘
                             │
              ┌──────────────┼──────────────┐
              ▼              ▼              ▼
        [미인증]         [인증완료]
              │              │
              ▼              ▼
       /login             /library  ◀──── 기본 진입점
                             │
               ┌──────────────┼──────────────┐
               ▼              ▼              ▼
           /library       /capture       /profile
               │
               └─▶ [상세검색/필터]
```

### 1.2 BottomTabs 구성 (전체 화면 공통 고정)

```text
┌────────────────────────────────────────┐
│  [Library]     [📷]     [Profile]      │  ← 하단 탭 내비게이션 (화면 하단 고정)
└────────────────────────────────────────┘
```

| 탭 | 아이콘 | 경로 | 비고 |
| --- | --- | --- | --- |
| Library | `grid-2x2` | `/library` | 기본 활성 탭 |
| Capture | `camera` | `/capture` | 중앙 FAB 스타일, 강조 색상 |
| Profile | `user` | `/profile` | 계정 정보 및 설정 통합 |

## 2. 화면별 UI 구성

### 2.1 Library Screen (`/library`)

에셋 전체 목록을 표시하는 메인 화면입니다.

```text
┌─────────────────────────────┐
│  [NOVA]                     │  ← 상단 바 (브랜드 로고)
│  ┌──────────────────────┐    │
│  │ 🔍 검색어... (X) [⊶] │    │  ← SearchBar (인라인 검색, 리셋(X) 및 필터 버튼)
│  └──────────────────────┘    │
│  [모두] [최근] [즐겨찾기]    │  ← 카테고리 필터 (기능별 칩셋)
├─────────────────────────────┤
│  ┌──────┐ ┌──────┐          │
│  │Asset │ │Asset │          │  ← AssetGrid (2-col, virtualized)
│  │Card  │ │Card  │          │
│  └──────┘ └──────┘          │
│  ┌──────┐ ┌──────┐          │
│  │Asset │ │Asset │          │
│  └──────┘ └──────┘          │
│           ...               │
├─────────────────────────────┤
│     [Library]     [📷]     [👤] │  ← BottomTabs
└─────────────────────────────┘
```

**주요 화면 구성 요소:**

| 요소 | 시각적 특징 및 역할 |
| --- | --- |
| 상단 바 | 서비스 로고를 좌측에 배치하며, 클릭 시 홈(라이브러리)으로 이동 |
| 통합 검색 바 | 텍스트 입력 시 실시간 인라인 필터를 수행하며, 리셋(X) 버튼으로 초기화 가능. 우측 상세검색 버튼([⊶]) 클릭 시 전용 검색 화면으로 전환 |
| 카테고리 필터 | '최근', '즐겨찾기' 등 목적에 따른 에셋 그룹핑을 제공하며 가로 스크롤로 탐색 가능 |
| 에셋 그리드 | 수집된 에셋들을 2열 격자 형태로 노출하며, 대용량 목록에서도 끊김 없는 스크롤 환경 제공 |
| 에셋 카드 | 썸네일과 대표 색상(팔레트), 관련 태그를 요약하여 에셋의 성격을 직관적으로 전달 |
| 하단 탭 내비게이션 | 라이브러리, 캡처, 프로필(설정 포함) 간 이동을 담당하며 항상 고정 |

### 2.2 Capture Screen (`/capture`)

카메라 촬영 및 파일 선택을 통해 에셋을 수집하는 화면입니다.

#### 1) 기본 카메라 화면 (Camera Mode)

사용자가 카메라 사용을 승인했을 때 표시되는 기본 촬영 상태입니다.

```text
┌─────────────────────────────┐
│ [══════════════] 업로드 진행 │  ← 진행 바 (저장 시에만 상단 노출)
│                      [✕]   │  ← 닫기 버튼 (라이브러리로 복귀)
│                             │
│  ┌─────────────────────┐   │
│  │                     │   │
│  │   실시간 미리보기    │   │  ← 뷰파인더 영역 (전체 화면 대응)
│  │                     │   │
│  │  ┌── 촬영 가이드 ──┐  │   │  ← 중앙 조준선 및 코너 안내선 오버레이
│  │  └────────────────┘  │   │
│  └─────────────────────┘   │
│                             │
│  ┌─────┐  ┌──────┐  ┌─────┐│
│  │ 🖼️  │  │  ⚪  │  │ �  ││  ← 하단 컨트롤바
│  │파일선택│  │ 셔터 │  │전환 ││     · 셔터: 카메라 촬영용 링 버튼
│  └─────┘  └──────┘  └─────┘│
│                             │
│  [플래시 설정]  [타이머 설정] │  ← 보조 옵션바

```

#### 2) 파일 선택/확인 화면 (File selection & Review)

갤러리에서 사진을 선택했거나, 촬영 결과물을 확인하는 상태입니다.

```text
┌─────────────────────────────┐
│                      [✕]   │
│  ┌─────────────────────┐   │
│  │                     │   │
│  │  선택된 사진 미리보기  │   │  ← 전체 화면 이미지 프리뷰
│  │                     │   │
│  │       [다시 선택]    │   │  ← 프리뷰 중앙 버튼
│  └─────────────────────┘   │
│                             │
│  ┌─────┐  ┌──────────┐ ┌──┐│
│  │ 🖼️  │  │  ⬆ 업로드 │ │🔄││  ← 촬영 버튼이 '업로드' 버튼으로 전환
│  └─────┘  └──────────┘ └──┘│
└─────────────────────────────┘
```

#### 3) 카메라 접근 불가 시 폴백 (Error States)

기기 사양이나 권한 문제로 카메라를 사용할 수 없을 때의 화면 구성입니다.

| 발생 상황 | 시각적 특징 | 안내 메시지(예시) |
| --- | --- | --- |
| 비보안 연결 | 카메라 차단 아이콘 | "보안 연결(HTTPS) 환경에서만 카메라를 사용할 수 있습니다." |
| 권한 거부 | 이미지 파일 아이콘 | "카메라 접근 권한이 없습니다. 갤러리에서 선택해 주세요." |
| 장치 오류 | 이미지 파일 아이콘 | "카메라를 찾을 수 없거나 지원되지 않는 기기입니다." |

#### 저장 프로세스 (User Experience Flow)

1. **조준**: 실시간 미리보기를 보며 가이드 프레임에 대상을 맞춥니다.
2. **동작**: 셔터 버튼을 누르거나 미리보기된 사진을 업로드합니다.
3. **처리**: 상단 진행 바가 채워지며 에셋 정보가 생성됩니다.
4. **확인**: 화면 하단에 "에셋이 추가되었습니다"라는 완료 알림이 나타납니다.
5. **복귀**: 알림 종료 후 다시 실시간 미리보기 상태로 전환되어 즉시 다음 촬영이 가능해집니다.

**주요 구성 요소:**

| 요소 | 시각적 특징 및 역할 |
| --- | --- |
| 실시간 뷰파인더 | 기기 방향에 관계없이 전체 공간을 채우는 카메라 프리뷰 |
| 촬영 가이드 | 정확한 구도를 위한 조준선과 모서리 가이드 선 |
| 가변 셔터 버튼 | 상황에 따라 '카메라 촬영' 혹은 '파일 업로드' 형태로 자동 전환 |
| 진행 상태 바 | 에셋 저장 절차를 시각적으로 보여주는 상단 바 |
| 완료 알림(Toast) | 저장 성공 여부를 전달하는 체크 아이콘 포함 플로팅 UI |

### 2.3 Search Screen (`/search`)

라이브러리의 검색 필드(텍스트 영역) 또는 상세검색 버튼([⊶])을 통해 진입하며, 태그, 컬러, 파일명을 아우르는 상세 검색 및 필터링 기능을 제공합니다.

```text
┌─────────────────────────────┐
│  [NOVA]                     │  ← 상단 바 (라이브러리와 동일한 쉘 유지)
│  ┌──────────────────────┐    │
│  │ 🔍 [검색어 입력...] (X) │    │  ← SearchBar (자동 포커스, 상세필터 버튼 숨김)
│  └──────────────────────┘    │
│  최근: #brand #ui #icon     │  ← RecentSearchChips
├─────────────────────────────┤
│  추천 컬러 검색              │
│  🔴 🔵 🟡 🟢 ⚫ ⚪          │  ← ColorPaletteFilter
├─────────────────────────────┤
│  검색 결과 (실시간)          │
│  ┌──────┐ ┌──────┐          │
│  │Asset │ │Asset │          │  ← AssetGrid (검색 결과)
│  └──────┘ └──────┘          │
├─────────────────────────────┤
│     [Library]     [📷]     [👤] │  ← BottomTabs
└─────────────────────────────┘
```

**주요 화면 구성 요소:**

| 요소 | 시각적 특징 및 역할 |
| --- | --- |
| 실시간 검색 바 | 라이브러리와 동일한 위치에 배치되어 시각적 일관성을 유지하며, 실시간 결과 반영 및 취소 기능 제공 |
| 최근 검색어 | 사용자가 이전에 검색했던 이력을 칩(Chip) 형태로 나열하여 빠른 재검색 지원 |
| 컬러 팔레트 필터 | 주요 색상 아이콘을 선택하여 해당 색상을 포함한 에셋들만 선별하는 시각적 필터링 제공 |
| 검색 결과 그리드 | 검색 조건에 맞는 에셋들을 라이브러리와 동일한 2열 격자 형태로 표시 |
| 결과 없음 안내 | 검색 결과가 없을 때 명확한 시각적 안내와 함께 다른 검색어 유도 메시지 노출 |

## 3. 공통 UI 규칙

### 3.1 Safe Area 대응

iOS의 Dynamic Island 및 홈 인디케이터(Home Indicator)와의 간섭을 방지하기 위해 다음 레이아웃 규칙을 적용합니다:

- **하단 내비게이션 (BottomTabs):** 기기의 `safe-area-inset-bottom` 만큼 하단 여백(Padding)을 추가하여 인터랙션 영역을 보호합니다.
- **캡처 컨트롤 (CaptureControls):** 기본 하단 여백(16px)에 `safe-area-inset-bottom`을 합산하여 셔터 버튼이 홈 인디케이터에 가려지지 않도록 배치합니다.

### 3.2 로딩 상태 규칙

| 상황 | 처리 방식 |
| --- | --- |
| 에셋 목록 로딩 | `AssetCard` Skeleton (Pulse animation) |
| 이미지 로딩 | Blur-up (저해상도 → 고해상도 전환) |
| 업로드 진행 | 상단 `UploadProgressBar` (0→100%) |
| AI 분류 중 | `AIShimmerOverlay` (shimmer 효과) |

### 3.3 제스처 인터랙션

| 제스처 | 화면 | 동작 |
| --- | --- | --- |
| Swipe-down | `BottomSheet` | 시트 닫기 |
| Swipe-down | `CaptureViewfinder` | 캡처 화면 닫기 |
| Long-press | `AssetCard` | 다중 선택 모드 진입 |
| Pinch-zoom | `AssetPreview` | 이미지 확대/축소 |

## 4. 수락 기준 (Acceptance Criteria)

### 공통 및 성능 (Core & Performance)

- [ ] **[내비게이션 고정]** 모든 최상위 경로(`/library`, `/capture`, `/profile`)에서 `BottomTabs`가 하단에 고정 표시된다. (이동 편의성)
- [ ] **[스크롤 성능]** `AssetGrid`는 100개 이상의 에셋 데이터에서도 `60fps` 이상의 성능을 유지한다. (쾌속 탐색)
- [ ] **[검색 인터랙션]** `SearchBar` 입력 후 `200ms debounce`를 거쳐 실시간 검색 결과가 갱신된다. (즉각적 피드백)
- [ ] **[제스처 닫기]** `BottomSheet`는 `swipe-down` 제스처에 반응하여 부드럽게 닫힌다. (자연스러운 조작)
- [ ] **[Safe Area 대응]** iPhone의 `Dynamic Island` 및 `Home Indicator` 영역이 Safe Area 인셋으로 정상 보호된다. (레이아웃 안정성)

### 촬영 및 저장 (Capture & Storage)

- [x] **[카메라 스트림]** HTTPS 환경에서 `getUserMedia`를 통해 실시간 카메라 영상이 뷰파인더를 채운다. (생생한 촬영 경험)
- [x] **[장치 전환]** 전/후면 카메라 전환 버튼 클릭 시 1초 이내에 스트림이 전환된다. (창작 유연성)
- [x] **[보안 폴백]** 비보안(HTTP) 환경에서는 `INSECURE` 에러 안내와 함께 갤러리 선택 버튼이 노출된다. (안전한 안내)
- [x] **[권한 폴백]** 카메라 접근 거부 시 `PERMISSION_DENIED` 에러와 함께 폴백 UI가 표시된다. (명확한 상태 인지)
- [x] **[미리보기 확인]** 사진 촬영 또는 갤러리 선택 후 뷰파인더에 결과물이 표시되며 셔터가 '업로드' 버튼으로 자동 전환된다. (업로드 의사 확인)
- [x] **[업로드 피드백]** 셔터 클릭 시 `UploadProgressBar`가 0→100% 진행되며 완료 후 `Toast` 알림이 나타난다. (신뢰감 있는 저장)
- [x] **[초기화 및 연속성]** 저장 완료 알림 후 `clearPreview()`가 트리거되어 즉시 다음 촬영이 가능한 상태로 복구된다. (연속 촬영 워크플로우)
- [ ] **[데이터 즉시성]** 업로드 완료 후 3초 이내에 `AssetGrid`에 새 카드가 노출된다. (결과물 즉시 확인)
- [ ] **[오프라인 대응]** 네트워크 단절 시 `IndexedDB` 대기열에 저장되며, 온라인 복구 시 즉시 자동 업로드를 시도한다. (데이터 유실 방지)
