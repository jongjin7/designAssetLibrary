# 공통 페이지 구성 명세 (Common Page Specification)

모바일과 데스크탑 플랫폼에 관계없이 동일한 사용자 경험과 브랜딩을 제공해야 하는 공통 페이지(로그인, 에러)에 대한 규칙입니다.

---

## 1. 로그인 및 OAuth 화면 (Login & OAuth Screen)

### 1.1 인증 제공자 가이드 (Auth Providers)
디자이너 및 비개발자 사용자의 접근성을 최우선으로 하여 다음 소셜 서비스를 지원합니다.

| 제공자 | 브랜드 컬러 (Hex) | 아이콘/로고 사용 규칙 |
| :--- | :--- | :--- |
| **Google** | `#FFFFFF` | 구글 공식 'G' 로고와 함께 테두리가 있는 화이트 버튼 사용. |
| **Kakao** | `#FEE500` | 카카오 브랜드 옐로우 배경에 블랙 텍스트/아이콘 사용. |
| **Naver** | `#03C75A` | 네이버 그린 브랜드 컬러 배경에 화이트 텍스트/아이콘 사용. |
| **Facebook** | `#1877F2` | 페이스북 블루 브랜드 컬러 배경에 화이트 텍스트/아이콘 사용. |

### 1.2 OAuth UI 인터랙션 규칙
- **버튼 디자인**: 모든 소셜 로그인 버튼은 `NVButton` 규격에 맞춰 높이 48px(모바일) / 52px(데스크탑)로 통일합니다.
- **로딩 상태**: 소셜 버튼 클릭 시 해당 버튼 내부에 `NVSpinner`를 노출하고 다른 버튼들은 비활성화(Disabled) 처리합니다.
- **분리선**: 소셜 로그인 영역 상단에 "또는 소셜 계정으로 시작하기"와 같은 문구와 함께 옅은 회색 분리선(`border-slate-800`)을 배치합니다.

### 1.3 개발 구현 가이드 (Development Details)
- **인증 엔진**: **Supabase Auth**를 기본 인증 엔진으로 사용합니다.
- **Redirect URI**: 
    - 개발: `http://localhost:3000/auth/callback`
    - 데스크탑 앱: `nova-app://auth/callback` (Deep link 대응)
- **보안 처리**: 
    - `PKCE (Proof Key for Code Exchange)` 메커니즘을 사용하여 클라이언트 보안을 강화합니다.
    - 인증 성공 후 세션 정보는 안전한 HttpOnly 쿠키 또는 브라우저 스토리지를 통해 관리합니다.

## 2. 에러 및 예외 화면 (Error & Exception Pages)

### 2.1 구성 및 레이아웃 (Layout Rules)
모든 에러 상황은 시스템 중심이 아닌 **사용자 복구 중심**으로 구성합니다.
- **Template**: `NVErrorLayout` (기능적 레이아웃) + `NVErrorView` (시각적 콘텐츠)
- **상태 코드 가이드**:
    - **404**: "길을 잃으셨나요?" 메시지와 함께 라이브러리 홈으로 유도.
    - **500**: "시스템에 짧은 휴식이 필요합니다." 메시지와 함께 새로고침 유도.
    - **Offline**: "연결이 끊겼습니다." 메시지와 함께 재시도(Retry) 버튼 노출.

### 2.2 반응형 인터페이스 규격
- **중앙 정렬**: 모든 해상도에서 `flex items-center justify-center`를 사용하여 콘텐츠를 뷰포트 정중앙에 배치합니다.
- **배경 처리**: 에러 상황에서도 NOVA 고유의 다크 테마 그라데이션과 글라스모피즘 효과를 유지하여 "안정적인 실패 경험"을 제공합니다.
- **접근성(A11y)**: 에러 상황 발생 시 스크린 리더가 즉시 에러 메시지를 읽어줄 수 있도록 `role="alert"` 또는 `aria-live="assertive"` 속성을 적용합니다.
