에이전트가 NOVA 프로젝트에서 코드를 커밋하고 푸시할 때 다음 단계를 엄격히 준수해야 합니다. 특히 `docs/` 디렉토리의 변경사항이 있는 경우 강제성이 적용됩니다.

## 1. 변경 사항 분석
- `git status`를 통해 변경된 파일 목록을 확인합니다.
- 변경된 파일 중 `docs/` 디렉토리에 포함된 파일이 있는지 식별합니다.

## 2. 문서화 규칙 검증 (docs 변경 시 필수)
`docs/` 디렉토리의 파일이 포함된 경우, 다음 체크리스트를 통과해야 합니다:
- [ ] `docs/CONTRIBUTING_DOCS.md` 파일을 다시 읽고 최신 규칙을 확인합니다.
- [ ] 각 수정된 `.md` 파일의 **H1 제목**이 한글/영문 병기 스타일인지 확인합니다 (예: `# 제목: 부제 (Title)`).
- [ ] 제목 바로 아래에 **한글 요약문**이 1~3문장 포함되어 있는지 확인합니다.
- [ ] 하위 기술 명세 섹션이 **영문**으로 작성되어 있는지 확인합니다.

## 3. 커밋 메시지 구성
`.gitmessage` 템플릿의 형식을 따릅니다:
- **형식**: `<type>(<scope>): <subject>`
- **Type**: `docs`, `feat`, `fix`, `chore`, `refactor`, `style`, `test`
- **Scope (docs 필수)**: `design`, `feat`, `setup`, `arch`
- **Subject**: 한글로 간결하게 작성합니다.

## 4. 커밋 및 푸시 실행
// turbo
1. `git add .` 명령으로 모든 변경사항을 스테이징합니다.
2. 위에서 구성한 메시지로 `git commit -m "[메시지]"`를 실행합니다.
3. 원격 저장소가 설정된 경우 에이전트에게  `푸쉬`를 요청하면 `git push`를 실행합니다.
