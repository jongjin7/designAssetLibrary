# 커밋 및 푸쉬 워크플로우 (Commit & Push Workflow)

이 워크플로우는 변경된 코드를 커밋하고, **사용자가 명시적으로 '푸쉬(Push)'를 요청할 때만** 원격 저장소에 반영합니다.

## 1. 개요

- 모든 커밋은 [CONTRIBUTING.md](../../docs/CONTRIBUTING.md)의 커밋 컨벤션을 따라야 합니다.
- 기본적으로 변경 사항을 `git add` 및 `git commit` 하며, `push`는 선택적 단계입니다.

## 2. 절차

1. **상태 확인 및 스테이징**
   - 변경된 파일 목록을 확인합니다.
   - `git add .` 명령어로 모든 변경 사항을 스테이징합니다.

2. **커밋 메시지 작성**
   - 아래 컨벤션에 맞춰 **한글로 요약**하여 작성합니다:
     - `docs: <한글 설명>` (문서 수정)
     - `feat: <한글 설명>` (새 기능)
     - `fix: <한글 설명>` (버그 수정)
     - `refactor: <한글 설명>` (코드 리팩토링)
   - 예시: `docs: 원자적 분할 가이드 추가 및 README 업데이트`
   - **원칙**: 작업 상세 내용(Subject)은 반드시 한글로 작성합니다.

3. **커밋 실행**
   - `git commit -m "<메시지>"`를 실행합니다.

// turbo
4. **원격 저장소 푸쉬 (선택 사항)**

   - **사용자가 "푸쉬해줘" 또는 "push"라고 명시적으로 명령한 경우에만** 이 단계를 실행합니다.
   - 명령: `git push`

---

## 3. 주의 사항

- 사용자가 단순히 "커밋해"라고 하면 3단계까지만 완료합니다.
- 사용자가 "커밋하고 푸쉬해"라고 하면 4단계까지 모두 완료합니다.
- 문서 링크가 상대 경로인지, H1 제목이 병기되었는지 다시 한번 확인하십시오.
